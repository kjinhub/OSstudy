# 데커 알고리즘 발표 자료

React와 Vite로 만든 30분 발표용 슬라이드입니다. 총 21장으로 구성되어 있으며, 계좌 입금 예시를 통한 상호배제 문제, 데커 알고리즘의 등장 배경, 1~4번 실패 버전, `flag`와 `turn`의 역할, 단계별 실행 추적, Peterson 알고리즘 비교, 현대 시스템에서의 한계를 포함합니다.

## 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173/`을 엽니다.

## 조작

- 다음 슬라이드: `→` 또는 `Space`
- 이전 슬라이드: `←`
- 인쇄 화면 열기: `P`
- 발표 모드 진입: 우측 상단 `발표 시작` 버튼 또는 `F`
- 발표 모드 종료: 우측 상단 `나가기` 버튼 또는 `ESC`
- 특정 슬라이드 바로 열기: `http://localhost:5173/?slide=16`
- 발표 모드로 바로 보기: `http://localhost:5173/?slide=1&present=1`
- 16번 슬라이드의 실행 추적 도구: 실행, 초기화, 이전 단계, 다음 단계, 처음으로, 끝으로 버튼

## 발표 구성

- 1-7장: 상호배제와 계좌 입금 경쟁 상태 예시
- 8-12장: 1~4번 실패 버전과 비교 정리
- 13-16장: 데커 알고리즘 핵심 구조와 Python Tutor 스타일 실행 추적
- 17-21장: 장점, 단점, Peterson 비교, 현대 대안, 최종 정리

## 실행 추적 컴포넌트

- `CodeEditor`: 데커 예제 코드를 입력하고 실행/초기화하는 영역
- `CodeEditor`: 현재 실행 중인 코드 줄을 코드 내부에서 하이라이트하는 영역
- `VariablePanel`: 현재 변수 값과 변경된 변수를 보여주는 영역
- `MemoryPanel`: `flag` 배열과 임계 구역 상태를 박스 형태로 보여주는 영역
- `ConsolePanel`: `console.log` 결과를 표시하는 영역
- `ExecutionControls`: 처음/이전/다음/끝 단계 이동 버튼
- `StepTimeline`: 전체 실행 단계 위치 표시
- `ExplanationPanel`: 현재 단계에서 일어난 일을 자연어로 설명하는 영역
